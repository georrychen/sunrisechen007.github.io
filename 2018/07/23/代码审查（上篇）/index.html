前言

本篇文章主要介绍 OCLint 的基本概念，如何使用 OCLint 自带的规则在 XCode 中审查代码.

OCLint是什么？

它是一个静态代码分析工具，可以检查 C、C++ 和 OC代码的质量。它有一套自带的代码规范，你可通过修改这些规则来达到规范代码的目的，比如你可以修改规则LONG_LINE， 来规定函数和参数列表的长度不可以超过100个字符，那么Xcode在编译时，会把不符合这个规则的代码在 XCode 中标记出来，以到达代码审查的作用，你也可以把统计结果以 html 的格式输出。 具体的规则，可组合选择，灵活应用

OCLint 自带扫描规则约 70 种，具体可见官网.
http://docs.oclint.org/en/stable/rules/index.html

下图图是一些常用的规则，具体如何使用，会在 XCode 配置脚本文件中详细介绍.

OCLint 怎么安装？

它有两种安装方式，一种是通过 Homebrew 安装，方便简单，缺点是不能自定义审核 规则；另一种是 下载源代码安装，步骤略复杂，但是我们可编写审核规则，将在下篇中介绍。

1. 通过 Homebrew 安装

打开终端输入如下三个命令，等待几分钟即可

安装完成后，重启终端，输入如下命令检查是否安装成功

source ~/.bash_profile
cd ${SRCROOT}
xcodebuild clean
xcodebuild | tee xcodebuild.log
oclint-xcodebuild
oclint-json-compilation-database \
-e Vendor \
-e Pods \
– \
-disable-rule ObjCAssignIvarOutsideAccessors \
-rc=LONG_VARIABLE_NAME=20 \
-rc=LONG_LINE=200 \
-max-priority-1 100000 \
-max-priority-2 100000 \
-max-priority-3 100000 \
-report-type xcode

`

-e Vendor \ : 不扫描这个 Vendor 文件夹里的内容
-disable-rule ObjCAssignIvarOutsideAccessors \ : 表示忽略这个规则
rc=LONG_LINE=200 \ : 表示遵守 LONG_LINE 这个规则，并限定值为200

PS:

  • 上面的脚本内容，不能有空格，有注释，不然编译结果会标红（规则不起作用），不标黄，不知道啥原因。
  • 编译结果报告除了可在 XCode 中输出，也可以以网页的形式显示结果，只需要把 -report-type xcode 修改 -report-type html -o oclintReport.html 即可编译完成后生一个网页文件
  • 每次编译前需要 clean 一下

附录

下面是一些常见规则脚本代码：

`python
-disable-rule ObjCAssignIvarOutsideAccessors \

#switch - case 的最小数目，默认为 3
-rc=MINIMUM_CASES_IN_SWITCH=3 \

#参数名字符限制，默认 20
-rc=LONG_VARIABLE_NAME=200 \

#忽略，变量名最小字符数限制默认3
-disable-rule ShortVariableName \

#循环嵌套数限制，默认10
-rc=CYCLOMATIC_COMPLEXITY=10 \

#类行数限制，默认 1000
-rc=LONG_CLASS=7 \

#每行的字符限制，默认 100
-rc=LONG_LINE=200 \

#方法行数限制，默认 50
-rc=LONG_METHOD=80 \

#连续未注释行数限制
-rc=NCSS_METHOD=10 \

#block嵌套层数限制，默认 5
-rc=NESTED_BLOCK_DEPTH=5 \

#类成员的限制，默认20
-rc=TOO_MANY_FIELDS=20 \

#类方法的限制，默认30
-rc=TOO_MANY_METHODS=30 \

#参数个数限制，默认10
-rc=TOO_MANY_PARAMETERS=10 \
